hexo+GitHub Pages设置

第一步：准备工作（环境安装）

在开始之前，确保你的电脑上已经安装了以下两个工具：

  1. Node.js（提供 Hexo 运行环境）
  2. Git（用于将代码推送到 GitHub）

安装完成后，打开终端（Windows 用户使用 Git Bash 或 CMD，Mac 用户使用 Terminal），运行以下命令检查是否安装成功：

第二步：在 GitHub 上创建仓库

  1. 登录你的 GitHub 账号。
  2. 点击右上角的 +，选择 New repository。
  3. 关键步骤：仓库名称（Repository Name）必须填写为： 你的GitHub用户名.github.io
  4. 设为 Public（公开）。
  5. 不要勾选 “Add a README file”，直接点击 Create repository。

第三步：本地安装与初始化 Hexo

在电脑上找一个专门存放博客的文件夹（例如 D:\Blog 或 ~/Blog），在终端中进入该文件夹：

初始化完成后，可以本地预览一下效果：

打开浏览器访问 http://localhost:4000，看到 Hexo 默认的欢迎页面就说明本地搭建成功了。按 Ctrl + C 可以关闭本地服务。

第四步：配置一键部署到 GitHub

为了让 Hexo 能够自动把打包好的静态页面推送到 GitHub，我们需要配置部署插件。

  1. 安装部署插件
    在你的博客根目录下（my-blog 文件夹内）运行：

Bash
npm install hexo-deployer-git –save
2. 修改配置文件
打开博客根目录下的 _config.yml（注意：是根目录下的，不是主题文件夹下的）。
滚动到最底部，找到 deploy 配置项，修改为以下内容：

YAML
deploy:
type: git
repo: git@github.com:你的GitHub用户名/你的GitHub用户名.github.io.git
branch: main
💡 小提示：

如果你电脑配置了 SSH Key，repo 填 git@github.com:…（推荐，免密输入）。

如果使用的是 HTTPS，repo 填 https://github.com/你的GitHub用户名/你的GitHub用户名.github.io.git。

GitHub 现在的默认主分支是 main，如果你的仓库默认分支是 master，请将 branch 改为 master。

YAML 语法严格，每个冒号 : 后面必须有一个空格！

第五步：生成并部署
配置完成后，只需要简单的“三板斧”命令，就能把博客发布到线上了：

Bash

1. 清理缓存（可选，建议每次部署前执行）

hexo clean

2. 生成静态文件

hexo generate (或简写 hexo g)

3. 部署到 GitHub Pages

hexo deploy (或简写 hexo d)
注：你可以把后两个命令组合使用：hexo g -d

如果是第一次使用 Git 部署，终端可能会提示你输入 GitHub 的用户名和密码（或 Token），或者要求验证 SSH 权限，按照提示操作即可。

第六步：查看你的网站
部署成功后，稍等 1~2 分钟（GitHub Pages 解析需要一点时间）。
在浏览器中输入 https://你的GitHub用户名.github.io，就能看到你的专属个人博客了！

🚀 日常写博客的常用命令：
hexo new “文章标题” ：创建一篇新文章（在 source/_posts 目录下生成 .md 文件）。

hexo g ：写完后生成静态页面。

hexo s ：在本地预览效果。

hexo d ：一键推送到线上。

如果在配置过程中遇到报错（比如 Git 权限、YAML 格式错误等），可以把错误信息发出来，我们一起解决！
